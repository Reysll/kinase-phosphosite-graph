Progress Notes — Kinase-Substrate HGN Prediction Pipeline
==========================================================
Last updated: 2026-05-26 (post-power-outage verification; four-network redesign completed)


CONTEXT
-------
This document summarises the technical changes made to the prediction pipeline
(src_prediction/) since the project began. The goal is a clear record of what
changed, why, and what the current numbers mean.


1. LEAVE-ONE-OUT: MODEL TRAINING REDESIGN
------------------------------------------
Background from email (round 1):
  "for one edge that is removed, you train on all the rest of data and then
  predict for that one site. Then next site..."

What was wrong before:
  Node2vec was re-run from scratch on every single trial (~29 min/trial).
  50-trial run took ~24 hours.

What was changed:
  Node2vec runs exactly once before the trial loop, on the full graph.
  Each trial only re-trains logistic regression (seconds). Embedding, training
  matrix, and scoring cache are all computed once and shared across all trials.

Result:
  50-trial run: ~24 hours → ~3 minutes.
  6,909-trial run: not feasible → ~5–10 minutes.


2. FEATURE COMPUTATION SPEEDUP
--------------------------------
Pair feature computation now uses matrix operations (numpy stacking of K and S
embedding matrices). Previously was a Python loop. Faster, same result.


3. NODE2VEC WALK GENERATION SPEEDUP
-------------------------------------
Custom walk functions added that bypass node2vec's O(nodes × degree²) transition-
probability precomputation:
  - p=q=1 (standard DeepWalk): uniform random walk, no precomputation.
  - p≠q: on-the-fly biased walk using np.isin on integer index arrays.

Current node2vec parameters (all four run scripts):
  dimensions:   32
  walk_length:  10
  num_walks:    25
  workers:       4  (Word2Vec training threads)
  p, q:        1.0  (DeepWalk mode — fast path)

LOO parallelism: n_jobs_outer=8 (ThreadPoolExecutor, shares memory).
Previously 6; bumped to 8 on 2026-05-26 — stalling was not caused by worker
count but by lack of progress prints.


4. EMBEDDING LEAKAGE FIX  ← KEY DESIGN DECISION
--------------------------------------------------
Background from email (round 2):
  "when you have a link between the kinase and phosphosite, and calculate
  node2vec, it kind of keeps that link information in the embedding. Even when
  you remove that edge from your training, there is still leakage (indirect)."

Fix implemented in leave_one_out.py → _build_embedding_relations():
  All PhosphoSitePlus KSA edges ('phosphorylates') are removed from the graph
  BEFORE node2vec runs. They are kept as training labels only.
  Non-PSP KSA edges from other sources are kept even if they duplicate a PSP
  edge, because the label being predicted is specifically the PSP annotation.

Effect on results:
  Performance dropped compared to pre-fix numbers (as expected — the model
  can no longer indirectly read the answer via embedding structure).
  The post-fix numbers are the scientifically correct ones.

  Pre-fix (leaked)  → Post-fix (correct)
  Generic held_out mean rank:   140.0 → 191.9
  Liver FC held_out mean rank:  148.8 → 177.3
  Generic best_true median:      39.0 → 78.0
  Liver FC best_true median:     63.0 → 83.0

  After fixing leakage, liver FC OUTPERFORMS generic on both metrics
  (lower mean rank and lower median rank). The previous "generic wins on
  multi-kinase hub sites" finding was partly an artifact of leakage
  benefiting well-connected hub sites in the generic graph.


5. ADJUSTED HELD-OUT RANK  ← CORRECT PRIMARY METRIC
------------------------------------------------------
Background from email:
  "if other true kinases ranked high, just drop them from ranking, and then
  calculate the ranks. e.g. site has k1/k2/k3, held out k1. scoring gives
  [k7, k4, k2, k1]. Drop k2 → [k7, k4, k1]. Rank of k1 = 3, not 4."

Implemented in leave_one_out.py → _compute_adjusted_held_out_rank():
  Before measuring the held-out kinase's rank, all OTHER known true kinases
  for that site are removed from the scored list. This isolates the model's
  ability to rank the specific held-out kinase, without penalising it for
  co-kinases that rightly score higher (their edges were not held out).

  adjusted_held_out_rank is now the primary per-trial rank metric.
  held_out_kinase_rank (unadjusted) is kept for backward compatibility.
  best_true_kinase_rank (best rank of any true kinase) also still reported.


6. FOUR-NETWORK DESIGN
-----------------------
Background from email:
  "construct 4 networks: one generic (no correlation), one control-specific
  (correlation from healthy samples), one liver-specific (correlation from
  cancer samples), one liver FC-based (correlation among FC values)."

Implementation:
  Liver_network_edges.csv.gz now contains ALL THREE correlation edge types.
  Each run script selects the relevant subset via allowed_relations.
  No need to rebuild the graph per experiment.

  Network          | Edges used
  -----------------+------------------------------------------
  Generic          | GENERIC_BASE_RELATIONS only
  Control          | + site_corr_ctrl_pos / site_corr_ctrl_neg
  Cancer           | + site_corr_cancer_pos / site_corr_cancer_neg
  Liver FC         | + site_corr_fc_pos / site_corr_fc_neg

  Minimum sample count for correlation: correlation requires ≥6 samples
  (all three types have 18 samples, so this is satisfied for all).

PSP KSA edges are automatically excluded from the embedding graph regardless
of which correlation type is selected (leakage fix, section 4).


7. MODULAR EMBEDDING STRATEGY
-------------------------------
Background from email:
  "please make your code modular, so if we have time, we can just change the
  embedding methods."

embedding_strategy.py defines:
  EmbeddingStrategy (ABC): abstract base with .fit(graph) → Dict[str, ndarray]
  Node2VecStrategy (dataclass): current implementation, wraps fit_node2vec_embeddings

To swap in a new embedding method (e.g. graph contrastive learning, spectral):
  1. Implement EmbeddingStrategy.fit()
  2. Pass an instance to run_leave_one_out(embedding_strategy=...)
  No changes to leave_one_out.py required.

Note: the email mentions degree-sensitive embedding as a limitation of node2vec
for hub sites. Graph contrastive learning with spectral filtering would reduce
this bias. Not implemented yet; the modular structure is in place.


8. PER-KINASE TOP-k% ANALYSIS
-------------------------------
Background from email:
  "For every kinase, let's report in what percentage of substrates it is ranked
  in top-1, top-5, and top-10 in generic, healthy, and liver FC."

Implemented in analyze_results.py → analysis_per_kinase_topk():
  Uses adjusted_held_out_rank (other co-kinases dropped before measuring).
  Only kinases with ≥5 trials in any network are included.
  Output sorted by liver FC top-10% descending.

  Interpretation (from email):
  - High % in both generic and liver → likely hub/degree bias, not liver-specific
  - High % in liver but not generic → genuinely liver-context-boosted kinase
  - High % in generic but not liver → liver context dilutes signal for that kinase

10. TOP-1 KINASE FREQUENCY — DEGREE BIAS FINDING
--------------------------------------------------
This is the most striking result: the correlation-based networks are dominated by
one or two kinases getting top-1 prediction on virtually ALL trials.

  Network     | Top-1 monopoly
  ------------+-----------------------------------------------
  Generic     | ULK1 (76.5%), WNK1 (21%), ZAP70 (2.5%)
  Control     | VRK2 (94.8%), YES1 (4.8%)
  Cancer      | VRK3 (95.5%), VRK2 (3.8%)
  Liver FC    | ULK1 (100.0%) — ALL 6,909 trials

Interpretation: This is exactly the node2vec degree bias the supervisor predicted.
Adding correlation edges makes certain kinases so highly connected (VRK2/VRK3 in
raw-correlation networks, ULK1 in FC network) that they dominate the embedding
and win top-1 everywhere regardless of biology.

Even when a network has BETTER mean/median rank, the top-1 prediction is almost
always wrong because a hub kinase monopolizes the top slot. The improvement in
rank metrics reflects non-hub kinases moving to ranks 20-50 (better than 150-200
in generic) rather than any kinase reaching rank 1.

This finding directly supports the supervisor's suggestion to use less
degree-sensitive embedding (graph contrastive learning with spectral filtering)
to remove this hub bias.


11. TTK CASE STUDY
-------------------
TTK (Mps1 kinase, mitotic regulator, liver-cancer relevant), 21 trials:

  Network     | held_out mean | VRK monopoly?
  ------------+---------------+--------------
  Generic     | 247.7         | ULK1 (18/21), WNK1 (3/21) — always wrong
  Control     | 25.7          | VRK2 (21/21) — hub wins top-1 but TTK ranks ~25
  Cancer      | 112.7         | VRK3 (21/21) — similar pattern
  Liver FC    | 242.6         | ULK1 (21/21) — similar to generic

Control dramatically improves TTK rank (mean 25 vs 248 in generic) even though
VRK2 always wins top-1. This means healthy co-expression structurally connects
TTK to its substrates much better than either the generic or FC graph.


12. RANKING SHIFTS (GENERIC vs LIVER FC)
-----------------------------------------
Most IMPROVED kinases in Liver FC (all 100% improved, mean delta):
  CSK (+241, 6 trials)   STK24 (+238, 8)   STK11 (+216, 15)
  MAP3K5 (+177, 10)      RET (+172, 15)

Most WORSENED kinases in Liver FC (all 0% improved):
  PKN1 (-177, 10)   MAP3K11 (-168, 7)   MAP3K8 (-156, 23)
  SIK2 (-155, 5)    PDK1 (-149, 5)

Note: In Liver FC, the top-1 prediction is ULK1 for 100% of all 6,909 trials.
The "improvements" for CSK/STK24/STK11 reflect the true kinase moving from
rank ~400 in generic to rank 25-160 in liver FC, even though ULK1 still wins
top-1. The "worsenings" for PKN1/MAP3K11/MAP3K8 reflect kinases that were
reasonably ranked in generic (rank 40-170) but collapse to rank 220-375 in
liver FC because their correlation structure is disrupted by the FC edges.


13. WHAT COMES NEXT
--------------------
1. Address the degree/hub bias via modular embedding swap. Architecture is ready
   (EmbeddingStrategy ABC). Candidate: graph contrastive learning with spectral
   filtering, or random-walk-with-restart using personalized PageRank.
2. Biological validation: NTRK2, MAPK7, MAP3K5, PRKCQ are the most improved
   kinases in Liver FC — look for liver cancer evidence in the literature.
3. TTK: explore why healthy correlations (Control, rank 25) work so much better
   than FC (rank 243). This may indicate TTK substrates are highly correlated
   in normal liver tissue but show divergent FC patterns in tumors.
4. Abstract writing: four-network comparison with the hub-bias caveat.
5. Progress timing fix: run_in_parallel now calls progress_fn from inside
   as_completed, so elapsed time and ETA are live (not post-hoc).
