Progress Notes — Kinase-Substrate HGN Prediction Pipeline
==========================================================
Last updated: 2026-05-21 (updated with 50-trial results and multi-kinase evaluation)


CONTEXT
-------
This document summarizes the technical changes made to the prediction pipeline
(src_prediction/) since our last discussion. The goal is to give you a clear
picture of what was changed, why, and how the system behaves now compared to
before, so you can give feedback before we run the full evaluation.


1. LEAVE-ONE-OUT: MODEL TRAINING REDESIGN
------------------------------------------
Background from your email:
  You confirmed the correct LOO logic: "for one edge that is removed, you train
  on all the rest of data and then predict for that one site. Then next site..."

What was wrong before:
  The pipeline was re-running node2vec from scratch on every single trial. That
  means for a 50-trial evaluation, node2vec was being fit 50 separate times on
  a graph of 17,736 nodes and ~2 million edges. Each fit was taking roughly
  29 minutes, which is why the full 50-trial run was taking ~24 hours.

  The only thing that differed between any two trials was which single
  phosphorylation edge was held out. Everything else — the graph structure,
  the embeddings, the candidate kinases, the negative pairs — was identical.

What was changed:
  Node2vec now runs exactly once before the trial loop begins, on the full graph.
  The resulting embedding dictionary (one vector per node) is shared across all
  50 trials. Each trial only re-trains the logistic regression model, which takes
  a few seconds. The LOO logic itself is unchanged: the held-out kinase-site pair
  is removed from the training labels (not from the graph) before fitting the
  model for that trial, then the site is scored against all candidate kinases.

Why the graph is not modified per trial:
  With 17,736 nodes and 2 million edges, removing one edge changes the
  embeddings by a negligible amount. More importantly, removing one
  phosphorylation edge from the graph per trial while leaving all the other
  phosphorylation edges in was providing inconsistent signal anyway — the
  embedding was still "seeing" all the other known kinase-site relationships
  through the graph structure. The cleaner and faster approach is to use the
  full graph for embedding and only control for the held-out pair at the
  label level.

Result:
  The same 10-trial debug run that was taking hours now completes in under a
  minute. The mean and median held-out ranks from the new run are essentially
  identical to the previous results (mean 146.6 vs 146.4, median 116.5 vs 118.0),
  confirming that the redesign does not change the methodology in a meaningful way.


2. FEATURE COMPUTATION SPEEDUP
--------------------------------
What was wrong before:
  The function that converts node embeddings into pair features (the input to the
  logistic regression model) was iterating over each kinase-site pair one row at
  a time using a Python loop. For a training set with thousands of pairs and 256
  features per pair, this was unnecessary overhead per trial.

What was changed:
  The feature computation now uses matrix operations. All kinase embeddings are
  stacked into a single matrix K, all site embeddings into S, and the four feature
  blocks [K, S, |K-S|, K*S] are computed in a single numpy call. The result is
  the same — it is just computed much faster.


3. NODE2VEC WALK GENERATION SPEEDUP
-------------------------------------
What was wrong before:
  The node2vec library precomputes second-order transition probabilities for every
  pair of adjacent edges in the graph before generating any walks. On a graph with
  2 million edges and an average node degree of ~118, this precomputation step is
  the main bottleneck — it was estimated to take 3+ hours even for a single fit.
  This was the reason the 50-trial run took 24 hours (50 fits × ~29 min each).

What was changed:
  Two custom walk generation functions were added that bypass this precomputation:

  - For p=1, q=1 (standard random walk / DeepWalk): walks are generated by
    uniformly sampling a random neighbor at each step. No precomputation at all.
    This is what we currently use.

  - For p≠1 or q≠1 (biased node2vec): weights are computed on-the-fly at each
    step using vectorized numpy operations (np.isin on integer index arrays),
    rather than precomputing and storing them for every possible edge triple.
    This avoids the upfront cost entirely while still producing correct biased
    walks.

  In both cases the walk generation is now O(num_walks × nodes × walk_length),
  which for our current settings (25 walks, 10 steps, 17,736 nodes) takes seconds.

Current parameter settings (run_baseline_similarity.py):
  - dimensions:   32
  - walk_length:  10
  - num_walks:    25
  - workers:       8  (was 1 — safe to raise now that embedding runs only once)
  - p:           1.0
  - q:           1.0

  Note: 8 workers are used for the Word2Vec training step (which parallelizes
  well). Previously workers was kept at 1 because each of the 50 parallel trial
  processes would have spawned its own workers — that constraint no longer applies.


4. FULL 50-TRIAL RESULTS: GENERIC vs LIVER COMPARISON
------------------------------------------------------
Both the generic model and the liver model were run on the same 50 frozen trials
so the comparison is fair. Both runs completed in under 5 minutes total (compared
to the previous ~24 hours for a single 50-trial run).

Results (50 trials, 420 candidate kinases):

                          Generic     Liver (80% FC corr)
  Mean held-out rank:     157.4       118.2
  Median held-out rank:   118.5       104.0
  Top-1 accuracy:         0%          0%
  Top-5 accuracy:         0%          4%  (2/50)
  Top-10 accuracy:        0%          6%  (3/50)
  Top-20 accuracy:        2% (1/50)   8%  (4/50)
  Rank improved in liver: —           29/50 trials

Key observations:
- The liver-context graph improves the held-out kinase rank in 29 of 50 trials.
- Mean rank improves by ~25% (157 → 118), median by ~12% (118.5 → 104).
- The top-1 prediction changed in all 50 trials, meaning the liver-specific
  fold-change correlation edges are meaningfully reshaping the model's output,
  not just making marginal adjustments.
- The "best true" and "held-out" metrics are identical because all 50 randomly
  sampled trials happened to have only one known kinase per site. This is expected
  given that 7,687 out of 10,146 sites (76%) have only one known kinase.

Important note on the frozen trial sample:
  The 50 frozen trials were sampled randomly from all positive edges, so they
  mostly contain single-kinase sites. This means the best_true_rank metric
  provides no additional information over held_out_rank in this sample. A
  dedicated multi-kinase evaluation is needed to test the more biologically
  interesting cases (see section 5 below).


5. MULTI-KINASE SITE EVALUATION (IN PROGRESS)
----------------------------------------------
Background from your email:
  "We can first check the sites with multiple kinases and check the biology of
  them. This may be faster."

Dataset breakdown:
  Total positive edges:        14,596
  Unique sites:                10,146
  Sites with exactly 1 kinase:  7,687  (76%)
  Sites with 2+ kinases:        2,459  (24%)
  Edges from multi-kinase sites: 6,909
  Sites with 10+ known kinases:    ~65  (including AKT1-S473 x19, TP53-S15 x15,
                                         GSK3B-S9 x20, PRKAA2-T172 x20)

Why multi-kinase sites matter:
  For a site with, say, 15 known kinases, holding out one and asking "does the
  model recover it?" is a strict and arguably unfair test. The more meaningful
  question is: does the model rank ANY known true kinase at top-1 or top-5? This
  is captured by the best_true_rank metric, which becomes informative only when
  a site has multiple known kinases. If the model predicts a different kinase than
  the one held out, but that predicted kinase is also a known true kinase for that
  site, that is a correct prediction — not a miss.

  For the 50-trial random sample above, best_true_rank = held_out_rank for every
  trial because all sampled sites had only one known kinase. The multi-kinase
  evaluation is where those two metrics will diverge and show the model's real
  biological signal.

Results (6,909 trials across 2,459 multi-kinase sites, avg 3.93 true kinases/site):

  Random baseline (expected by chance with avg 3.93 kinases out of 420):
    Top-1: 0.9%   Top-5: 4.6%   Top-10: 9.1%   Top-20: 17.5%

                          Generic     Liver (80% FC corr)
  held_out mean rank:     140.0       148.8
  held_out median rank:   107.0       125.0

  best_true top-1:        0.1%        1.0%
  best_true top-5:        2.9%        2.9%
  best_true top-10:       12.9%       8.9%         (random: 9.1%)
  best_true top-20:       29.3%       22.8%        (random: 17.5%)
  best_true mean rank:    64.1        76.3
  best_true median rank:  39.0        63.0

Key finding — unexpected reversal:
  On multi-kinase sites, the generic model outperforms the liver model on the
  best_true_rank metric, which is the opposite of what the 50-trial random
  evaluation showed. This is interpretable:

  Multi-kinase sites are dominated by well-studied hub phosphosites (AKT1-S473
  with 19 known kinases, GSK3B-S9 with 20, TP53-S15 with 15, PRKAA2-T172 with
  20, etc.). These sites are already richly connected in every database the
  generic graph uses — PPI, pathway membership, coevolution. The generic graph
  captures that established signal cleanly.

  The ~1 million liver fold-change correlation edges reflect co-expression
  patterns in liver tumor tissue. For well-known hub sites this appears to
  dilute the core kinase-substrate signal rather than add to it, because the
  added edges connect based on tissue-level co-regulation, not direct kinase-
  substrate biology.

  The two results together form a consistent story:
  - Liver graph is better for liver-specific / less well-studied phosphosites
    (50-trial general sample: mean rank 118 vs 157 for generic).
  - Generic graph is better for hub sites already covered richly by databases
    (multi-kinase: best_true median rank 39 vs 63 for liver).

  This is actually a useful finding for the abstract: liver-specific context
  adds the most value precisely where the generic graph is sparse — which is
  what you would expect and hope from a context-specific graph.

  The generic model clearly beats the random baseline at top-10 (12.9% vs 9.1%)
  and top-20 (29.3% vs 17.5%). The liver model beats random only at top-20
  (22.8% vs 17.5%) and is essentially at the random level at top-10 (8.9% vs
  9.1%), suggesting the correlation edges are not helping for these sites.
