PROJECT PIPELINE EXPLANATION
============================
Last updated: 2026-05-27

Overview
========
This project builds and evaluates a context-aware biological network for kinase-substrate
association (KSA) prediction, with a specific focus on liver cancer.

The pipeline has two main layers:

1. Graph construction
   Builds biological networks from known data sources (generic) and extends them with
   liver-specific context (three correlation edge types).

2. Prediction and evaluation
   Uses the graph to rank candidate kinases for phosphosites under leave-one-out evaluation,
   comparing four network variants: generic, control (healthy), cancer (tumor), and liver FC.

Full flow:
1. Build generic graph
2. Build liver-specific graph (all three correlation types in one file)
3. Freeze multi-kinase trial set (reproducible evaluation)
4. Run four LOO experiments (generic / control / cancer / liver FC)
5. Run analysis (four networks side-by-side, per-kinase top-k%, TTK, ranking shifts)

SECTION 2. GRAPH CONSTRUCTION LAYER
=====================================

File: src/builders/kinase_substrate.py
---------------------------------------
Reads PhosphoSitePlus (PSP) Kinase_Substrate_Dataset.tsv.
Creates:
  PROTEIN:GENE nodes for kinases and substrates
  SITE:GENE_pSXXX nodes for phosphosites
  has_site edges: substrate protein -> phosphosite
  phosphorylates edges: kinase -> phosphosite  (PSP KSA = prediction target)

Filters: human-only rows, exactly one site token per row.
These PSP 'phosphorylates' edges are the prediction target throughout the pipeline
and are EXCLUDED from node2vec embedding to prevent leakage (see Section 3).


File: src/builders/ppase_substrate.py
--------------------------------------
Reads phosphatase-substrate table. Creates dephosphorylates edges.
Analogous to kinase_substrate.py; adds mechanistic context.


File: src/main.py
------------------
Orchestrates the generic graph build:
1. Load PSP + phosphatase data
2. Merge nodes and edges
3. Add node metadata (is_kinase, is_phosphatase, protein_role)
4. Add PTMsigDB site-pathway co-membership edges (site_same_pathway)
5. Add MsigDB protein-pathway co-membership edges (protein_same_pathway)
6. Add PPI edges (ppi_high_confidence, from STRING or equivalent)
7. Add PTMcode2 coevolution edges (site_coevolution)
8. Write outputs/nodes.csv.gz, outputs/edges.csv.gz

Output: ~13,318 nodes, ~1.1 million edges.


File: src/liver_network.py
---------------------------
Extends the generic graph with liver cancer context.

Steps:
A. Add liver-observed proteins and phosphosites (from LiverCancer_ProtExp_Phospho_casecntrl.xlsx)
   New nodes gain pathway/PPI/coevolution edges by re-running integration on the expanded node set.

B. Build three correlation edge types (all computed in one pass, stored in one file):

   site_corr_fc_pos / site_corr_fc_neg
     Pearson correlation of tumor/mean_control fold-change vectors across 18 tumor samples.
     Captures how phosphosite regulation changes together in tumors vs baseline.

   site_corr_ctrl_pos / site_corr_ctrl_neg
     Pearson correlation of raw abundance vectors across 18 healthy/control samples.
     Captures stable baseline co-expression structure in normal liver tissue.

   site_corr_cancer_pos / site_corr_cancer_neg
     Pearson correlation of raw abundance vectors across 18 tumor samples.
     Captures co-expression in the disease state (raw, not normalised).

   Threshold: 80th percentile for all three types. Min samples: 18 (>=6 satisfied).
   Edges stored in outputs/Liver_network_edges.csv.gz alongside all other edge types.
   Each LOO run script selects the relevant subset via allowed_relations.

C. Protein co-expression: DISABLED (only 3 usable rows after preprocessing).

Output: ~16,605 nodes, ~2.1 million edges.


File: src/run_liver.py
-----------------------
Launcher for liver_network.py with experiment settings:
  site_corr_percentile = 80.0
  add_protein_fc_corr  = False


SECTION 3. PREDICTION AND EVALUATION LAYER
============================================

Key design decisions (all implemented):

1. NODE2VEC RUNS ONCE, ON A PSP-STRIPPED GRAPH (LEAKAGE FIX)
   PSP 'phosphorylates' edges are removed from the graph before node2vec runs.
   Embeddings therefore reflect structural/pathway context only — not KSA relationships.
   The same embedding is shared across all 6,909 trials; only logistic regression is
   re-trained per trial (seconds, not hours).
   Non-PSP KSA edges from other resources are kept even if they duplicate a PSP edge.

2. ADJUSTED HELD-OUT RANK (PRIMARY METRIC)
   Before measuring the held-out kinase's rank, all OTHER known true kinases for that
   site are removed from the scored list.
   Example: site has k1/k2/k3, k1 held out. Scored: [k7, k4, k2, k1].
            Drop k2 -> [k7, k4, k1]. adjusted_held_out_rank of k1 = 3 (not 4).
   Column: adjusted_held_out_rank. Legacy column held_out_kinase_rank kept for compat.

3. MODULAR EMBEDDING STRATEGY
   src_prediction/embedding_strategy.py defines:
     EmbeddingStrategy (ABC): fit(graph: nx.Graph) -> Dict[str, np.ndarray]
     Node2VecStrategy (dataclass): current implementation (p=q=1, DeepWalk mode)
   To swap: implement EmbeddingStrategy.fit() and pass to run_leave_one_out(embedding_strategy=).

4. FOUR NETWORKS, ONE LIVER GRAPH FILE
   allowed_relations controls which edge types enter the embedding graph and training:
     Generic: GENERIC_BASE_RELATIONS (no correlation edges)
     Control: + site_corr_ctrl_pos/neg
     Cancer:  + site_corr_cancer_pos/neg
     Liver FC: + site_corr_fc_pos/neg

5. PARALLEL PROGRESS REPORTING
   parallel_utils.run_in_parallel() accepts progress_fn(done, total) callback.
   Callback fires inside as_completed() loop — elapsed and ETA are live.
   n_jobs_outer=8 (ThreadPoolExecutor). Node2Vec Word2Vec workers=4.


File: src_prediction/leave_one_out.py
--------------------------------------
Core LOO evaluation engine. One-time setup:
  1. Build embedding graph (PSP edges stripped)
  2. Fit embeddings via embedding_strategy.fit()
  3. Build training feature matrix (all positive + sampled negative pairs)
  4. Build scoring feature cache for all unique test sites

Per-trial (microseconds each after setup):
  1. Numpy mask to exclude held-out pair from training
  2. LogisticRegression.fit() on masked training set
  3. predict_proba() on pre-cached scoring features for the test site
  4. Compute held_out_kinase_rank, adjusted_held_out_rank, best_true_kinase_rank

Output columns (per trial):
  trial_index, true_kinase_node_id, site_node_id,
  held_out_kinase_rank, adjusted_held_out_rank, best_true_kinase_rank,
  n_true_kinases_for_site, all_true_kinases_for_site,
  top1_predicted_kinase, top1_score, num_candidate_kinases_scored


File: src_prediction/run_multi_kinase_{generic,control,cancer,liver}.py
------------------------------------------------------------------------
One script per network. Each:
  1. Loads the appropriate graph (generic or liver)
  2. Sets allowed_relations for that network variant
  3. Calls run_leave_one_out() with n_jobs_outer=8
  4. Writes results.csv.gz, metrics.csv.gz, summary.txt to outputs_prediction/<name>/

All four scripts use the same frozen_trials_multi_kinase.csv.gz (6,909 trials).


File: src_prediction/analyze_results.py
-----------------------------------------
Reads all four experiment results and produces supervisor-facing analysis:

  1. performance_summary.txt — held-out, adjusted, best-true rank for all four networks
  2. top1_frequency_summary.txt — which kinase wins top-1 prediction most often per network
  3. per_kinase_topk.csv / .txt — per-kinase top-1/5/10% using adjusted_held_out_rank
  4. ttk_deepdive.txt — TTK-specific metrics across all four networks
  5. ranking_shifts.txt — top-20 biggest improvements/worsenings (generic vs liver FC)

All outputs written to outputs_prediction/analysis/.


File: src_prediction/embedding_strategy.py
-------------------------------------------
Defines the EmbeddingStrategy ABC. Node2VecStrategy is the current concrete implementation.
Parameters: dimensions=32, walk_length=10, num_walks=25, workers=4, p=q=1 (DeepWalk).
The leave_one_out module imports only EmbeddingStrategy; any conforming implementation works.


File: src_prediction/parallel_utils.py
----------------------------------------
run_in_parallel(items, worker_fn, max_workers, use_threads, progress_fn)
  - ThreadPoolExecutor (use_threads=True): threads share memory — no numpy array copying.
    sklearn liblinear releases the GIL during fitting -> real parallelism.
  - ProcessPoolExecutor (use_threads=False): for Linux cluster (fork = copy-on-write).
  - progress_fn(done, total) fires inside as_completed() for live elapsed/ETA reporting.


File: src_prediction/relation_filters.py
-----------------------------------------
Defines the allowed-relation sets:
  GENERIC_BASE_RELATIONS: has_site, phosphorylates, dephosphorylates,
                          site_same_pathway, protein_same_pathway,
                          ppi_high_confidence, site_coevolution
  PSP_KSA_RELATIONS:      {phosphorylates}  <- always excluded from embedding graph
  SITE_CORR_FC_RELATIONS: {site_corr_fc_pos, site_corr_fc_neg}
  SITE_CORR_CTRL_RELATIONS: {site_corr_ctrl_pos, site_corr_ctrl_neg}
  SITE_CORR_CANCER_RELATIONS: {site_corr_cancer_pos, site_corr_cancer_neg}


SECTION 4. EVALUATION DATASET
===============================

Total positive edges (PSP KSA, human):   14,596
Unique phosphosites:                      10,146
Sites with exactly 1 known kinase:         7,687  (76%)
Sites with 2+ known kinases:               2,459  (24%)
Edges from multi-kinase sites:             6,909  <- frozen trial set
Unique candidate kinases:                    420

The frozen trial set (frozen_trials_multi_kinase.csv.gz) contains all 6,909 edges
from multi-kinase sites. All four LOO experiments use the same frozen set so
comparisons are fair.

Trial definition: one trial = one held-out kinase-site edge. Trials per kinase =
number of multi-kinase sites that have that kinase as one of their annotated kinases.


SECTION 5. CURRENT EXPERIMENTAL RESULTS
=========================================

All four networks completed (2026-05-27). Post-leakage-fix numbers.

Adjusted held-out rank (primary metric, lower = better):

                        Generic   Control   Cancer    Liver FC
  Mean rank:            190.4     177.7     186.0     175.8
  Median rank:          188.0     166.5     174.0     165.0
  Ranking:              4th       2nd       3rd       1st

Best-true rank (best rank any true kinase achieves):

                        Generic   Control   Cancer    Liver FC
  Mean rank:            114.6     103.2     117.0     108.6
  Median rank:           78.0      80.0     116.0      83.0

Key findings:
  - Cancer WORSE than Generic: raw tumor correlations add noise.
  - Control rivals Liver FC: stable healthy co-expression is nearly as informative.
  - ALL correlation networks show severe hub-kinase degree bias in node2vec (see below).

Top-1 prediction monopoly (hub bias):
  Generic:  ULK1 (76.5%), WNK1 (21%)
  Control:  VRK2 (94.8%)
  Cancer:   VRK3 (95.5%)
  Liver FC: ULK1 (100%)  <- every single trial

The EmbeddingStrategy swap architecture is in place to address this bias.


SECTION 6. END-TO-END EXAMPLE
===============================

Held-out edge: NTRK2 -> SITE:SHC3_pY218

  Generic result:   top-1 = VRK3,  held-out rank = 380
  Liver FC result:  top-1 = VRK2,  held-out rank = 63   (delta = +317)

The true kinase moved from rank 380 to rank 63 when liver fold-change context
was added — a genuine structural improvement. However VRK2 still wins top-1
due to hub degree bias. After the embedding swap, the expectation is that NTRK2
would move into the top-5 and occupy top-1 more often.


SECTION 7. SCIENTIFIC MEANING
===============================

Generic graph files:    establish prior biological knowledge
Liver network file:     adds disease-specific signaling context (three types)
LOO evaluation files:   test whether context changes kinase prioritisation
Analysis files:         make those differences inspectable and interpretable

Core scientific finding:
  Adding liver-specific correlation edges improves rank metrics (true kinase
  moves closer to the top) but node2vec's degree sensitivity concentrates
  top-1 predictions on hub kinases, obscuring biological signals. The
  methodological contribution is the modular embedding architecture that
  enables this bias to be corrected without changing the evaluation pipeline.
