PROJECT GOAL
=============
Study how disease-specific biological context changes kinase-substrate
association (KSA) prediction. Incorporate liver cancer context into a
heterogeneous biological graph and evaluate whether it changes kinase
prioritisation in biologically meaningful ways.

SECTION 3. PREDICTION MODEL — CURRENT DESIGN
=============================================
Two strategies implemented under the EmbeddingStrategy ABC:

  Node2VecStrategy (original):
    DeepWalk mode (p=q=1), dimensions=32, walk_length=10, num_walks=25.
    Suffers from degree-based hub bias — high-connectivity nodes dominate
    all top-1 predictions regardless of biology.

  SpectralEmbeddingStrategy (new, Phase 2):
    Normalized Laplacian: L = I - D^{-1/2} A D^{-1/2}.
    Degree-normalizing by construction: high-degree nodes penalized.
    directed=False (symmetric Laplacian required).
    scipy ARPACK eigsh, 32 eigenvectors, ~4 s on liver graph.
    Implemented instead of GCL (Dr. Ayati's suggestion) because spectral
    achieves the same degree-normalization with a closed-form solution.
    GCL still swappable via EmbeddingStrategy.fit() if needed.

Key design decisions (unchanged from Phase 1):
  - Embedding runs ONCE on PSP-stripped graph (leakage fix).
    PSP 'phosphorylates' edges removed from embedding; kept as training labels.
  - Training matrix, scoring cache built ONCE; each trial is LR mask + fit.
  - ThreadPoolExecutor (threads share memory), n_jobs_outer=8.

Primary metric: adjusted_held_out_rank
  Other known true co-kinases removed from scored list before ranking
  the held-out kinase. e.g. site has k1/k2/k3, k1 held out:
  scored [k7, k4, k2, k1] -> drop k2 -> [k7, k4, k1] -> rank of k1 = 3.

  Thresholds: poor <= 5 (166 kinases), average 5-20 (133), rich > 20 (121).
  Each site now returns one top-1 prediction per category.

  Key additional output columns:
    held_out_kinase_category          -- category of the held-out kinase
    adjusted_held_out_rank_in_category -- rank within category (not 420)
    top1_poor / top1_average / top1_rich

  Hub kinase placement:
    VRK3 (1 substrate)  -> poor
    VRK2 (15 substrates)-> average
    ULK1 (77 substrates)-> rich
  Cross-category wall confirmed working: within-category hubs are different
  kinases (WNK2, TGFBR2, YES1) from the global hub (ULK1). However,
  within-category hub bias still exists — that is what spectral targets.

Design decision: pre-filter counts
  Dr. Ayati left open "before or after multi-kinase filtering". We use counts
  from the full generic graph because richness reflects global PSP annotation,

  Network     | adj. median rank | top-1 always
  ------------+------------------+---------------
  Generic     |     245          | ULK1
  Control     |      25          | VRK2
  Cancer      |     111          | VRK3
  Liver FC    |     239          | ULK1

Phase 2 spectral results (adj. median rank, within-rich rank):
  Network     | adj. median rank | adj. cat rank | top-1 (global)
  ------------+------------------+---------------+----------------
  Generic     |       2          |      2        | IRAK4 (14 trials), TTK (7 trials)
  Control     |      14          |     10        | CDK1
  Cancer      |      14          |     10        | PIK3R1
  Liver FC    |      14          |     10        | PIK3R1

KEY FINDING (Phase 2, spectral):
  The GENERIC spectral graph ranks TTK at global median rank 2 out of 420.
  Every trial is either rank 1 (7 trials: non-TTK substrates, top-1=TTK)
  or rank 2 (14 trials: TTK autophosphorylation sites, top-1=IRAK4).
  TTK's structural position in the PPI/pathway graph is already informative.

  The correlation networks (control/cancer/liver FC) WORSEN TTK from rank 2
  to rank 14. Adding correlation edges shifts the embedding so that PIK3R1
  (liver hub) or CDK1 (control hub) dominate, pushing TTK down.

  Interpretation: TTK's liver cancer role (activating AKT) operates through
  a pathway-level mechanism NOT visible as cohesive substrate fold-change
  co-variation. The pathway structure alone (generic graph) captures TTK's
  substrate relationships better than any correlation-based addition.

13.1 ConcatEmbeddingStrategy
-----------------------------
Added to src_prediction/embedding_strategy.py. Concatenates a Node2VecStrategy
vector and a SpectralEmbeddingStrategy vector per node (32+32=64 dims with
both at default dimensions). directed=True: node2vec fits on the directed
graph it's handed directly (matching its standalone behaviour exactly);
spectral gets an explicit graph.to_undirected() copy, since each sub-strategy
needs its own appropriately-directed graph per Dr. Ayati's note, rather than
assuming one shared graph works for both.

13.2 Predict-all inference pass (src_prediction/inference_all.py + run_inference_all.py)
-------------------------------------------------------------------------------------------
Generalizes LOO from "evaluate only the held-out edge, only on multi-kinase
sites" to "score every (candidate kinase, target site) pair" — needed because
LOO structurally cannot produce a full kinase x site rank matrix.

Two-tier scoring to avoid leakage without an LOO-style trial per site:
  - Known true (kinase, site) edges (736 of them, after restricting the full
    14,596-edge liver-positive set to the 3,228 FC-valid sites): scored by a
    model with ONLY that one edge masked. Other true edges, including other
    co-kinases at the same site, stay in training.
  - Every other (kinase, site) pair: scored by ONE global model trained on
    all 736 true edges, nothing held out (there's nothing to leak for a pair
    that was never a positive training label).
Reuses leave_one_out.py's _build_training_matrix/_build_scoring_cache
unchanged; only the masking/assembly logic is new.

Network graph source: "generic" uses GENERIC_NODES/GENERIC_EDGES (matches the
existing LOO scripts), NOT the liver graph restricted to base relations.
Tried the latter first — it fragments the liver graph into many disconnected
2-node protein-site components (liver-only sites whose only edges are the
correlation ones being stripped), which hangs SpectralEmbeddingStrategy's
ARPACK eigensolver (near-zero eigenvalue multiplicity blows up under
unshifted "SM" mode). Reverted to the GENERIC file; confirmed working.
Consequence: "generic" only covers 391/3,228 (12%) of FC-valid sites — a
real, reportable asymmetry (its site population is essentially PSP-annotated
sites, much smaller than the broader liver-proteomics-detected set the other
three networks cover at 100%), not a bug.

Full 12-combo sweep (4 networks x 3 embeddings) ran in ~30 min total — far
cheaper than the Phase 1/2 LOO runs (186 min for 4 networks) because each
combo only needs 736 masked trials instead of 6,909.
Output: outputs_prediction/inference_all/{network}_{embedding}/ranks.csv.gz
(site_node_id, kinase_node_id, score, rank_in_site, is_true_kinase) +
summary.txt per combo.

13.4 Result 2 — per-kinase rank-distribution KS-test (build_rank_ks_test.py)
---------------------------------------------------------------------------
Two network pairs x three embeddings = 6 KS-test runs (scipy.stats.ks_2samp,
BH-FDR corrected):
  1. control vs liver_fc   (Dr. Ayati's original request)
  2. cancer vs control     (Salvador's "cleaner healthy-vs-disease" alternative)

CRITICAL FINDING during this build: most kinases' rank_in_site is nearly
CONSTANT across all 3,228 sites within one network (e.g. CDK20 under
control_spectral is rank 349 at literally every single site, zero variance).
This is the existing hub bias showing up in a new way — for non-true-kinase
pairs, a kinase's own embedding dominates its score far more than the site's
embedding does, so its rank barely depends on which site is being scored.
A KS-test on two such near-degenerate (point-mass-like) distributions that
differ by even one rank position gives ks_statistic ~= 1.0 trivially, which
is why a naive run flagged 412-414/415 kinases as "significant" — true in
the strict statistical sense, but not a meaningful biological finding.

Fix: every kinase's within-network rank std-dev is now reported
(std_rank_{network}), and an `informative` flag marks kinases where at least
one side has std_rank > 1.0 (rank genuinely varies by site). The summary's
highlighted list is restricted to informative + significant kinases; full
per-pair CSVs keep every kinase (informative and not) so degenerate cases
stay visible rather than hidden.

Informative-kinase counts (of 415 tested), by embedding:
  node2vec: ~408-410/415 informative — node2vec's local walk structure
    produces more site-varying scores even for non-hub kinases.
  spectral: ~263-264/415 informative — spectral's global community structure
    is much more degenerate; most kinases get a fixed per-network baseline.
  concat:   ~409/415 informative — node2vec's local signal dominates the
    informativeness even when concatenated with spectral.

Outputs: outputs_prediction/analysis/ks_test_{pair}_{embedding}.csv (all
kinases, all columns) and ks_test_summary.txt (informative+significant
highlights per pair/embedding).
